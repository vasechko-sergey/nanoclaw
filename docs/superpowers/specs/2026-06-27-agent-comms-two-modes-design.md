# Spec — Два режима связи для всех агентов

**Дата:** 2026-06-27
**Статус:** дизайн на ревью
**Тип:** instruction-only (правки CLAUDE.md + общего INSTRUCTIONS.md; кода нет)

## Проблема

Все 5 агентов (jarvis/payne/greg/gordon/scrooge) **уже** доступны человеку напрямую: в
приложении человек выбирает агента в левом дровере и пишет ему — это давно реализовано
(см. memory `ios-app-channel`). Но инструкции это не отражают единообразно:

- **Greg** написан как «ты **не** общаешься с человеком напрямую, твой единственный
  собеседник — Jarvis» (`groups/greg/CLAUDE.md:7`, ещё раз `:26`). Это противоречит
  реальности приложения и блокирует прямой режим (вопросы по здоровью, приём анализов,
  уточняющие вопросы — всё, что нужно для Спека 2 «диагност»).
- **Payne / Gordon / Scrooge** де-факто работают в обоих режимах (Payne ведёт тренировку
  с человеком + a2a с Грегом; Gordon — фото еды; Scrooge — траты + a2a-пинг Джарвису), но
  «два режима» нигде не сформулированы явно — каждый описывает только свою половину.
- **Jarvis** — хаб, но инструкции косвенно подразумевают, что здоровье и пр. идёт «через
  него». На деле специалист доступен и напрямую.

Нет единой модели: **кто с тобой говорит и как себя вести**.

## Модель: два режима

Каждый агент в каждом входящем сообщении находится в одном из двух режимов, **определяемых
по отправителю**.

**Детект (заземлён в `container/agent-runner/src/formatter.ts:233,243`):** каждое входящее
`<message>` несёт атрибут `from="<destination>"` (резолвится через `findByRouting`). Список
своих destination-имён агент видит в рантайм-аддендуме каждого хода.

- `from` = имя соседнего агента (jarvis/payne/greg/gordon/scrooge) → **командный режим (a2a)**.
- иначе (`from` = человеческий канал `ios-app-v2:*` / telegram, либо `unknown:*`) →
  **прямой режим**; `sender` = имя человека.

(Дискриминатор — именно `from`. `sender` у a2a часто `Unknown` — на него не опираться.)

| | Прямой (человек) | Командный (a2a/агент) |
|---|---|---|
| Адресат ответа | человек | агент-источник по контракту |
| Форма | живой диалог в своей персоне, простой язык | структурный payload (finding/signal/JSON) |
| Уточнения | можно спрашивать, идти туда-обратно | терминально: один ответ, без авто-пингов (hop-cap 5) |
| Гейт к человеку | сам | Jarvis (или принимающий агент) решает, что показать человеку |

**Jarvis = хаб, НЕ обязательный передатчик.** Человек может прийти к любому специалисту
напрямую. a2a — для кросс-домен пуша (Greg→Payne сигнал, Greg/Scrooge→Jarvis срочное), а не
«всё через Jarvis».

**Инвариант:** дисциплины, не зависящие от режима, остаются в обоих — §Factual discipline,
дисклеймеры (Greg «без диагнозов»), quiet hours, no-flattery, простой язык.

## Изменение 1 — общий `groups/INSTRUCTIONS.md`

Новая секция **«## Два режима связи»** сразу после §«Agent-to-agent (a2a)» (она ссылается на
механизм `from`, описанный там и в §Communication discipline). Содержимое — модель выше:
два источника, правило детекта по `from`, таблица поведения, «Jarvis — хаб не передатчик»,
инвариант дисциплин. Один абзац + таблица, без дублирования механики a2a (на неё ссылка).

## Изменение 2 — `groups/greg/CLAUDE.md` (главная правка)

**Строка 7** (личность) — убрать «не общаешься напрямую / единственный собеседник Jarvis»:

> Ты — Грег, автономный аналитик здоровья своего человека. Работаешь в **двух режимах**
> (см. INSTRUCTIONS §Два режима связи): **напрямую с человеком** — он пишет тебе в приложении
> (вопрос о здоровье, прислал анализы, разбор самочувствия → живой диалог в твоём голосе,
> можешь уточнять) — и **в команде** — a2a с Jarvis и Пейном (findings/signals, терминально).
> Числа всегда считаешь скриптом и трактуешь данные, а не память.

**Строка 26** (язык) — развести два режима:

> Язык — русский. В прямом режиме говоришь человеку сам; в командном — `house_quote`/
> `observation` попадают человеку через Jarvis-брифы verbatim. В обоих — простой русский,
> жаргон разворачивай: [таблица остаётся без изменений]

Остальное у Грега совместимо и остаётся: §Дисклеймер (без диагнозов — оба режима), §Манера
(House-голос), §Память, §Команда (a2a-контракты), §Recheck. Детальная прокачка differential
для прямого режима (диалоговый differential + уверенность) — **Спек 2**, здесь не трогаем.

## Изменение 3 — Payne / Gordon / Scrooge (лёгкая, единообразие)

Каждому в секцию «Команда» добавить одну строку-указатель:

> Каналы — два режима (напрямую с человеком + a2a), см. INSTRUCTIONS §Два режима связи.

Поведение у них уже корректно для обоих; правка делает модель явной и единой. Существующие
a2a-контракты не меняются.

## Изменение 4 — `groups/jarvis/CLAUDE.md` (лёгкая)

В §Команда — одна строка, что специалисты доступны человеку **и напрямую**, Jarvis не
обязан быть единственным каналом; его роль — оркестрация и бриф, а не релей по умолчанию.
Существующее поведение (читает профили, a2a-приём findings, гейт) не меняется.

## Non-goals

- Никакого кода. Только инструкции.
- Не меняем механику a2a, destinations, маршрутизацию, форматтер.
- Differential/уверенность/анализы/калибровка Грега — Спек 2.
- Не трогаем персоны, скилы (кроме строк-указателей в CLAUDE.md).

## Деплой

`groups/` gitignored → **scp** правленых CLAUDE.md + общего INSTRUCTIONS.md на VDS + chown.
Затем **rebirth** затронутых агентов (kill контейнер + DELETE continuation по всем сессиям —
стандарт для instruction-правок, иначе живая SDK-сессия их не перечитает; memory
`feedback_agent_instruction_reload`). INSTRUCTIONS.md общий → rebirth всех 5; CLAUDE.md
точечные → rebirth затронутого. Практически: правим всех → rebirth всех 5.

## Верификация

Инструкции → ручные пробы (нет юнит-тестов):

1. **Greg прямой режим** — человек пишет Грегу в приложении «почему я не выспался?» →
   Грег отвечает живым House-диалогом напрямую человеку (не finding-JSON, не «спроси Jarvis»).
2. **Greg командный режим** — Jarvis шлёт a2a recheck → Греg отвечает структурным finding'ом
   терминально (как раньше). Регрессии нет.
3. **Прочие** — выборочно: Payne стартует тренировку с человеком; Greg→Payne `health_signal`
   доходит. Существующие потоки целы.

Пробы выполняет Сергей после деплоя.
